# Grant Use Plan

## Current stage

The project is at an early but concrete stage. A small public prototype now exists with a synthetic data generator, a first interpretable baseline, a scoring pipeline, sample outputs, and written safety/scope documentation.

## What has already been built

- synthetic aggregate signal generation
- a robust baseline scoring method
- CSV and plot outputs for inspection
- a lightweight evaluation pass on synthetic labels
- public documentation covering scope, safety, and next steps

## Current bottlenecks

The main bottlenecks are not grand research claims. They are practical constraints that make it harder to turn a small prototype into a stronger research artifact:

- limited compute for broader but still safe benchmarking
- limited tooling for repeatable experiments and comparison across baselines
- limited infrastructure for reproducibility, packaging, and evaluation tracking
- no lightweight hosted interface for showing the workflow cleanly if that becomes useful

## Specific resources that would accelerate progress

Modest grant support would help with:

- compute for larger safe experiments on synthetic and appropriate public datasets
- project-specific software or tooling where justified
- lightweight hosting for a simple demo or dashboard if needed
- reproducibility and evaluation infrastructure such as better experiment logging, packaging, and automated checks

## Likely outputs with modest support

With modest support, the project could reasonably deliver:

- a stronger benchmarked prototype with several benign baseline methods
- clearer evaluation reports and uncertainty summaries
- a cleaner reproducible workflow for rerunning experiments
- improved documentation and usability for external review
- a more inspectable public research artifact suitable for discussion with collaborators or funders
